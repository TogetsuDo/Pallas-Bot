# 从 3.x 迁到 4.0

升级站点，或维护者把 `dev-v2` 合进 `main` 发 4.0 GA，看这页。

## 1. 站点从 3.x 升级到 4.0

### 1.1 升级前备份

- `config/pallas.toml`
- `data/`（含 `pallas_config/webui.json`）
- 数据库与协议端实例数据（若使用）

### 1.2 拉取 4.0 代码

```bash
git fetch origin
git checkout dev-v2   # GA 前集成线；发版后改为 main 或 v4.0.0 tag
uv sync --dev
```

按需 extras：`--extra pg`、`--extra coord-redis`、`--extra deploy-all`。

### 1.3 配置迁移

| 3.x | 4.0 |
| --- | --- |
| 根目录 `.env` 主配置 | `config/pallas.toml` + `webui.json` |
| `CHAT_ENABLE` / `OLLAMA_*` | `LLM_CHAT_ENABLED` 等，见 [ollama 迁移](llm-migrate-from-ollama.md) |
| 内置玩法插件 | 对应 `pallas-plugin-*` 官方扩展 |

一次性从旧 `.env` 迁移：

```bash
uv run python tools/migrate_env_to_pallas.py
```

### 1.4 安装官方扩展

原 3.x 内置能力需显式安装，例如：

```bash
uv run pallas ext install pallas-plugin-protocol
uv run pallas ext install pallas-plugin-duel
uv run pallas ext install pallas-plugin-draw
```

或通过 WebUI 插件商店安装。安装后重启 Bot。

### 1.5 验证清单

- [ ] `uv run nb run` 启动无 import 错误
- [ ] WebUI 可登录；未完成 setup 时进入 Setup Wizard
- [ ] 所需玩法扩展在 `uv run pallas ext list` 中为已安装
- [ ] 智能接话：Pallas-Bot-AI 4.0+ 已起且 `LLM_CHAT_ENABLED` 已开
- [ ] 分片站点：核对 `deploy/` 与 `REDIS_URL`（若使用）

## 2. 维护者：dev-v2 → main 合并

### 2.1 前置条件（P0 门禁）

合并前确认路线图 P0 项已 Review/Done，至少包括：

- OpenAPI 导出与 WebUI codegen（OPT-WEB-001/002）
- Setup / AI Wizard（OPT-WEB-003/004）
- 窄屏审计（OPT-WEB-005）
- LLM capability 信封与 AI health 单一事实源（OPT-LLM-001/002）
- `pallas-core` wheel 可构建（`./scripts/build_core.sh`）
- 本迁移指南与 [CHANGELOG](../../CHANGELOG.md) 已更新

### 2.2 推荐合并步骤

1. 冻结 `dev-v2`：跑全量 CI + 关键集成（Bot + WebUI + AI + 常用扩展）
2. 更新版本号：`pallas/__init__.py` `__version__`、发版 tag 草案
3. `git checkout main && git merge --no-ff dev-v2`（或 PR 经 review）
4. 打 tag `v4.0.0` 并发布 GitHub Release（附 CHANGELOG 摘要）
5. 可选：`./scripts/build_core.sh --publish` 发布 `pallas-core` 至 PyPI
6. 文档站与 Docker 镜像跟进 tag

### 2.3 合并后站点通知要点

- 3.x 不再兼容 `src/` 插件路径；社区插件需 `pallas.api.*`
- 默认 slim core；扩展需商店或 `pallas ext install`
- 配置以 `pallas.toml` + `webui.json` 为准

## 3. 已知限制（4.0 GA）

| 项 | 说明 |
| --- | --- |
| **分支** | 4.0 集成在 `dev-v2`，合入 `main` 随 GA tag（见 §2） |
| **pallas-core PyPI** | wheel 可本地构建（`./scripts/build_core.sh`）；PyPI 正式发布随 GA tag |
| **i18n** | WebUI 仍以中文为主（P2 OPT-WEB-020） |
| **向量记忆 / RAG** | 默认 hybrid（关键词 + embedding，依赖 AI 仓）；本地 `data/pallas_knowledge/` ingest；图谱 UI 仍在 backlog |
| **插件子进程隔离** | 未采用 MaiBot 式子进程（长期 backlog） |
| **streaming @聊天** | 非流式分段输出（P2 OPT-LLM-020） |

**已在 4.0.x 收口、不再视为 GA 阻塞：**

- OpenAPI 导出 + WebUI codegen 客户端（OPT-WEB-001/002）
- Setup Wizard + AI 体检向导（OPT-WEB-003/004）
- 窄屏 ≤560px 审计（OPT-WEB-005，见 WebUI 自检清单）
- LLM L2 capability / health / memory（OPT-LLM-001～003）
- 社区商店安装进度 SSE（OPT-ARCH-011）
- 日志 SSE Last-Event-ID（OPT-WEB-012）
- `/pallas/api/*` 异常错误包装（OPT-WEB-013）

问题排查见 [FAQ](../FAQ.md)、[升级（maintainer）](../maintainer/deploy/upgrade.md)。
