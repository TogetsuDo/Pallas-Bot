# 从 3.x 迁到现行版本

从 3.x 升级已有站点，按下面步骤做。新装看 [五分钟跑起来](quickstart.md)。

## 第 1 步：先备份

- `config/pallas.toml`
- `data/`（含 `pallas_config/webui.json`）
- 数据库与协议端实例数据（若使用）

## 第 2 步：拉代码并装依赖

```bash
git fetch origin
git pull
uv sync --dev
```

按需 extras：`--extra pg`、`--extra coord-redis`、`--extra deploy-all`。  
也可按 [Release](https://github.com/PallasBot/Pallas-Bot/releases) 检出对应 tag。

## 第 3 步：配置对照

| 3.x | 现行 |
| --- | --- |
| 根目录 `.env` 主配置 | `config/pallas.toml` + `webui.json` |
| `CHAT_ENABLE` / `OLLAMA_*` | `LLM_CHAT_ENABLED` 等，见 [ollama 迁移](llm-migrate-from-ollama.md) |
| 内置玩法插件 | 对应 `pallas-plugin-*` 官方扩展 |

从旧 `.env` 一次性迁：

```bash
uv run python tools/migrate_env_to_pallas.py
```

## 第 4 步：装回官方扩展

3.x 里「自带」的能力，现在要显式装，例如：

```bash
uv run pallas ext install pallas-plugin-protocol
uv run pallas ext install pallas-plugin-duel
uv run pallas ext install pallas-plugin-draw
```

或走 WebUI 插件商店。装完重启 Bot。

## 第 5 步：验收

- [ ] `uv run nb run` 启动无 import 错误
- [ ] WebUI 可登录；未完成 setup 时进入 Setup Wizard
- [ ] 所需玩法扩展在 `uv run pallas ext list` 中为已安装
- [ ] 智能接话：Pallas-Bot-AI 已起且 `LLM_CHAT_ENABLED` 已开
- [ ] 分片站点：核对 `deploy/` 与 `REDIS_URL`（若使用）

升级后请注意：

- 社区插件需走 `pallas.api.*`（不再兼容旧 `src/` 插件路径）
- 默认 slim core；扩展需商店或 `pallas ext install`
- 配置以 `pallas.toml` + `webui.json` 为准

## 已知限制

| 项 | 说明 |
| --- | --- |
| **pallas-core PyPI** | 可本地构建（`./scripts/build_core.sh`）；正式包以发布为准 |
| **i18n** | WebUI 仍以中文为主 |
| **向量记忆 / RAG** | 默认 hybrid（关键词 + embedding，依赖 AI 仓）；本地 `data/pallas_knowledge/` ingest；图谱 UI 仍在规划 |
| **插件子进程隔离** | 未采用 MaiBot 式子进程 |
| **streaming @聊天** | 非流式分段输出 |

排障：[FAQ](../FAQ.md)、[升级（maintainer）](../maintainer/deploy/upgrade.md)。
