# 从 3.x 迁到 4.0

从 3.x 升级到 4.0，**多数站点可以无痛完成**：保留 `config/`、`data/` 与数据库，用 CLI 拉齐代码与依赖、重启即可。新装请直接看 [五分钟跑起来](quickstart.md)。

## 你会动到什么

| 通常不用重做 | 记得对齐 |
| --- | --- |
| 数据库与群数据 | **WebUI** 静态资源（与 Bot 同版本） |
| 已装的官方插件（pip） | **Pallas-Bot-AI**（若用智能接话 / 唱歌等） |
| `local/plugins/` 里的站点插件 | 协议端实例（一般随 Bot 升级即可） |
| `webui.json` 里的控制台配置 | |

官方插件**不必因为升到 4.0 就重装一遍**：以前已在控制台商店或 pip 装过的会保留；`local/plugins/` 里有的同名优先加载；默认 `load_bundled_extra_plugins = "auto"` 时，没 pip 包也会用镜像内副本。只有**新站点**或**以前没开过的玩法**才需要去控制台 **插件商店** 补装（见 [安装官方插件](install-extensions.md)）。

## 推荐：CLI 升级 Bot，插件交给控制台

**Bot 与 WebUI** 用 CLI（SSH / 初次部署）；**官方插件** 装/卸/更新留在升级后登录控制台处理即可。

在 **Pallas-Bot 仓库根**执行（按需加 `--sync-extra`）：

```bash
uv run pallas maintenance run \
  --update-bot \
  --update-webui \
  --sync-extra pg \
  --dev
```

- `--update-bot`：`git pull` 对齐当前分支
- `--update-webui`：下载 Release 附带的 `dist.zip` 到 `data/pallas_webui`
- `--sync-extra`：按需带上 `pg`、`coord-redis`、`deploy-all` 等
- 默认在 Bot 更新后 **自动重启**；仅同步依赖不重启时加 `--no-restart`

使用 AI 媒体能力时，另在 AI 仓执行 `uv run pallas ai setup`（或 `git pull` + 该仓 bootstrap），见 [LLM 与 AI 运维](../maintainer/operate/llm-and-ai.md)。

升级后若要补装或更新官方插件：打开 **插件商店**（不必为插件单独跑 CLI）。

## 第 1 步：备份

- `config/pallas.toml`
- `data/`（含 `pallas_config/webui.json`）
- 数据库与协议端实例数据（若使用）

## 第 2 步：升级本体与 WebUI

**推荐（上文一条命令）**；或分步：

```bash
uv run pallas update bot
uv run pallas update webui
uv run pallas sync --dev --extra pg    # 按需替换 extra
uv run pallas restart
```

也可按 [Release](https://github.com/PallasBot/Pallas-Bot/releases) 检出对应 tag 后 `uv run pallas sync`。

## 第 3 步：配置（多数可跳过）

4.0 以 `config/pallas.toml` + `data/pallas_config/webui.json` 为准；`webui.json` 覆盖 `pallas.toml`。

| 3.x 习惯 | 4.0 |
| --- | --- |
| 根目录 `.env` 主配置 | `pallas.toml` + `webui.json`（已迁过可不动） |
| `CHAT_ENABLE` / `OLLAMA_*` | `LLM_CHAT_ENABLED` 等，见 [ollama 迁移](llm-migrate-from-ollama.md) |

若仍主要靠根目录 `.env`，可一次性迁移：

```bash
uv run python tools/migrate_env_to_pallas.py
```

迁移后 `.env` 仍可保留 nb/pip 插件专用项；与 `webui.json` 避免同名键重复。

## 第 4 步：验收

```bash
uv run pallas status
```

检查清单：

- [ ] 启动无 import 错误
- [ ] WebUI 可登录；未完成 setup 时进入 Setup Wizard
- [ ] 群里 **牛牛帮助** 与升级前能力一致
- [ ] 智能接话：`LLM_CHAT_ENABLED` 已开且 `uv run pallas ai setup --check-only` 通过
- [ ] 分片站点：`deploy/`、`REDIS_URL`、registry 仍一致（若使用）

## 少数需要留意的变化

- **社区插件**：须走 `pallas.api.*`，不再兼容旧 `src/plugins/` 直引内核路径。见 [社区插件作者指南](community-plugin-author.md)。
- **新装 vs 升级**：4.0 默认 slim core，**全新部署**才需要按需装官方插件；从 3.x 升上来且玩法已在跑，一般不用重复安装。
- **配置源**：运行时以 `pallas.toml` + `webui.json` 为准，勿再往根 `.env` 堆主配置项。

更完整的分层升级顺序见 [升级（maintainer）](../maintainer/deploy/upgrade.md) 与 [CLI 参考](../maintainer/reference/cli.md)。排障：[FAQ](../FAQ.md)。
