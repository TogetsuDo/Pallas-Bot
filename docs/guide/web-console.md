# 网页控制台

Bot 启动后，**Web 控制台会自动跟着起来**，不用单独开关。

::: tip 默认地址
| 项 | 值 |
| --- | --- |
| 路径 | `http://<主机>:8088/pallas/` |
| 端口 | `pallas.toml` 里 `[bootstrap] port`，默认 **8088** |
| 首次口令 | 看启动日志，或 `data/pallas_console/` |
:::

---

## 登录

1. 浏览器打开 `/pallas/`  
2. 输入日志里打印的 **控制台口令**  
3. 登录后可改配置、看日志、进插件商店  

::: warning 生产环境
不要长期开启 **开发模式**（会跳过登录）。公网访问请配 HTTPS + 强口令。
:::

**如何确认成功**：能看到首页运行概览、侧栏有「插件」「运行日志」等菜单。

---

## 常去哪些页面

| 想做什么 | 侧栏去哪 |
| --- | --- |
| 看牛是否在线、消息量 | **仪表盘** |
| 改某个插件的配置 | **插件** → 点插件名 |
| 装决斗、MAA 等 | **插件商店** |
| 关某插件（全群立即生效） | **插件** → 插件卡片 **全实例禁用** |
| 看报错日志 | **运行日志** / **日志报错** |
| 管 NapCat 多账号 | **协议端实例**（或 `/protocol/console/`） |
| 改 AI 地址、外部服务 | **通用配置** |
| 备份数据库 | **数据库** |

保存插件相关项后，写入 `data/pallas_config/webui.json`，**一般不用重启**；装/卸 pip 扩展包 **必须重启 Bot**。

---

## 和协议端页的区别

| 页面 | 地址 | 干什么 |
| --- | --- | --- |
| Web 控制台 | `/pallas/` | 配置、日志、插件商店 |
| 协议端管理 | `/protocol/console/` | 建 NapCat、扫码上 QQ |

两个页面用**同一套登录口令**。

---

## 配置存在哪

- 你在网页里点的「保存」→ `data/pallas_config/webui.json`  
- 只改端口、数据库、超管 → 编辑 `config/pallas.toml` 后 **重启 Bot**  

详见 [配置存储](../architecture/settings-storage.md)。
