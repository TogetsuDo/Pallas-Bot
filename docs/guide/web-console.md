# 网页控制台

Pallas-Bot 启动后，Web 控制台会跟着起来，不用单独开关。

::: tip 默认地址
| 项 | 值 |
| --- | --- |
| 路径 | `http://<主机>:8088/pallas/` |
| 端口 | `pallas.toml` → `[bootstrap] port`，默认 **8088** |
| 首次口令 | 启动日志，或 `data/pallas_console/` |
:::

## 登录

1. 打开 `/pallas/`  
2. 输入日志里的控制台口令  
3. 即可改配置、看日志、进插件商店  

::: warning 生产环境
不要长期开开发模式（会跳过登录）。公网请配 HTTPS + 强口令。
:::

能看到首页运行概览、侧栏有「插件」「运行日志」= 登录成功。

## 常去哪些页

| 想做什么 | 侧栏 |
| --- | --- |
| 看是否在线、消息量 | **仪表盘** |
| 改某个插件配置 | **插件** → 点插件名 |
| 装决斗、MAA | **插件商店** |
| 关某插件（全实例） | **插件** → **全实例禁用** |
| 看报错 | **运行日志** / **日志报错** |
| 管 NapCat | **协议端实例** 或 `/protocol/console/` |
| 改 AI / 外部服务 | **通用配置** |
| 备份库 | **数据库** |

保存插件配置写入 `webui.json`，**一般不用重启**。装 / 卸 pip 扩展 **必须重启 Bot**。

## 和协议端页的区别

| 页面 | 地址 | 干什么 |
| --- | --- | --- |
| Web 控制台 | `/pallas/` | 配置、日志、商店 |
| 协议端管理 | `/protocol/console/` | 建 NapCat、扫码 |

两页用**同一套口令**。

## 配置存在哪

| 你怎么改 | 落哪 |
| --- | --- |
| 网页点保存 | `data/pallas_config/webui.json` |
| 改端口 / 库 / 超管 | `config/pallas.toml`，然后 **重启** |

详见 [配置存储](../architecture/settings-storage.md)。
