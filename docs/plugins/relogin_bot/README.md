# relogin_bot

私聊里为指定 QQ **重启协议端并推送登录二维码**，或为超管 **创建新牛牛账号并启动**。依赖 **`pallas_protocol`** 已加载且协议端管理可用。目前只支持NapCat协议端

## 依赖

- **`pallas_protocol`**：账号、启停、协议端目录与二维码路径均由协议端插件维护。
- **`BotConfig` / 数据库**：校验「号主」（`admins`）、写入新牛牛的 `admins`。
- **私聊**：两条命令均需在 **私聊** 中使用；群内会提示「请私聊」。

## 命令

### `牛牛重新上号`

供 **目标牛牛的号主（`admins`）** 或 **超管** 使用。

1. 私聊发送 `牛牛重新上号`，可按提示输入 QQ；或直接 `牛牛重新上号 123456789`（纯数字 QQ）。
2. 若该 QQ 在协议端**已有账号**：重启进程并等待 `cache/qrcode.png` 更新，随后私聊发送二维码图片。
3. 若协议端**无账号**但数据库里**已有该 bot 配置**：会提示输入 **昵称**，自动 `create_account` 后再走重启与二维码流程。
4. 若数据库中也不存在该 QQ 的牛牛配置：直接结束并提示不存在。

输入 `取消` / `cancel` / `退出` / `quit` 可中断流程。

### `创建牛牛`

仅 **超管**（`SUPERUSER`）可用。

**一行参数**（私聊）：

```text
创建牛牛 <牛牛昵称> <牛牛QQ> <号主QQ> [更多号主QQ ...]
```

示例：`创建牛牛 我的小牛 3888888888 2777777777`

**分步交互**：私聊发送 `创建牛牛` 后按提示依次输入昵称、牛牛 QQ、号主 QQ（多个号主用空格分隔）。

流程：在协议端 `create_account` → `start_account` → 将号主写入该 bot 的 `admins` → 若有新二维码则私聊发送图片。

## 排障

| 现象 | 说明 |
|------|------|
| 「请私聊使用该命令」 | 必须在私聊窗口对 Bot 发命令。 |
| 不是某 QQ 的管理员 | 重新上号仅该牛牛的 `admins` 或超管可操作。 |
| 「60 秒内未检测到新的二维码」 | 协议端已启动但未刷新 `account_data_dir/cache/qrcode.png`；请到 **协议端管理页** 查看或查进程日志。 |
| 「账号目录缺失」 | 协议端账号数据目录未就绪，先检查 `pallas_protocol` 实例与磁盘路径。 |
| 创建/启动失败 | 看 Bot 日志中的异常信息；常见为协议端镜像/路径/端口冲突等。 |

实现见 [`src/plugins/relogin_bot/__init__.py`](../../../src/plugins/relogin_bot/__init__.py)。
